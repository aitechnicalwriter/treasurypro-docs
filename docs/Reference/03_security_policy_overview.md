# Security Policy and Contact Information

This overview summarizes TreasuryPro's policies on payment security, password management, and who to contact in case of suspicious activity.

## Security Requirements

| Policy Area | Requirement |
| :--- | :--- |
| **Authentication** | All users must utilize **Multi-Factor Authentication (MFA)** for login and payment approval. |
| **Password** | Must be at least 12 characters long and include numbers and special characters. Passwords expire every 90 days. |
| **Data Retention** | All transaction logs and audit reports are permanently stored for 7 years to meet regulatory compliance standards. |
| **Separation of Duties (SoD)** | The user who creates a payment cannot also be the final approver for that same payment. |

## Urgent Contact Information

In the event of suspected fraud, a security breach, or unauthorized account activity:

| Contact Role | Action | Contact Channel |
| :--- | :--- | :--- |
| **Internal Treasury Support** | Lock your user account and investigate initial details. | **Phone:** 1-800-QBANK-CS (Option 3) |
| **QBank Connect Security Team** | Review API logs and coordinate with banking partners. | **Email:** security@qbankconnect.com |
