This glossary defines technical terms and concepts frequently used within the TreasuryPro application and documentation.

| Term | Definition | Context |
| :--- | :--- | :--- |
| **ACH** | **Automated Clearing House.** An electronic network for processing transactions, usually low-value and non-urgent (e.g., payroll). | Payment methods |
| **Available Balance** | The amount of cash currently available for use. This includes all cleared transactions and funds the bank has made available. | Real-Time Cash Positioning |
| **BAI2** | **Bank Administration Institute, Format 2.** The standard file format banks use to send corporate balance and transaction statements. | Reconciliation |
| **Cutoff Time** | The deadline for submitting a payment (e.g., ACH batch) for same-day processing. | Payment processing |
| **Finality** | The legal point in time when a funds transfer is considered complete and irreversible (e.g., a wire transfer). | Payment risk |
| **Idempotency** | The property that ensures a payment request, even if submitted multiple times, will only be executed once. | Fraud prevention, API stability |
| **Ledger Balance** | The balance of an account at the start or end of a business day, reflecting transactions that have been fully posted. | Reconciliation |
